> The `@AGENTS.md` import above is load-bearing: **read https://docs.expo.dev/versions/v56.0.0/ before writing any Expo code.** APIs in this SDK band changed meaningfully (`expo-router` v6, `expo-glass-effect`, `expo-symbols`, `@expo/ui` beta) ‚Äî don't write from memory.

FPL Gaffer is an Expo / React Native (SDK 54, RN 0.81, React 19) app backed by Supabase, wrapping the public Fantasy Premier League API.

## Commands

```bash
npm start                 # expo start (dev server)
npm run ios | android | web
npm run lint              # expo lint
npm test                  # jest (jest-expo preset)
npx jest path/to/file.test.ts      # single file
npx jest -t "name substring"       # single test by name
```

Tests are only collected from `**/__tests__/**/*.test.ts(x)` (see `testMatch` in `package.json`) ‚Äî a `*.test.ts` outside an `__tests__/` dir is silently ignored. `src/__tests__/` mirrors the `src/` tree.

**Local gotchas (hit these once):**
- **jest hangs after you've run the dev server.** Running `npm start` (Metro) leaves watchman in a recrawl state that stalls a later `npx jest` on the haste-map crawl. Fix: `watchman shutdown-server` then run with `--watchman=false --runInBand --forceExit`. (CI is unaffected.)
- **`typedRoutes` staleness after adding/renaming a route** under `src/app/`. `.expo/types/router.d.ts` is regenerated by Metro, so `tsc --noEmit` flags the new pathname (and any `router.push` to it) until you start the dev server once. Tests are unaffected ‚Äî they mock `expo-router`.
- `npm run lint` (`expo lint`) auto-generates an untracked `eslint.config.js` if none exists (the repo has no committed ESLint config) ‚Äî don't commit it as part of unrelated work.

Supabase (local stack + edge functions in `supabase/`):

```bash
supabase start                          # local Postgres/Auth/Functions in Docker
supabase db push                        # apply migrations to linked project
supabase functions deploy fpl-ingest    # or: ping
./supabase/scripts/test-ingest-locally.sh
```

CI (`.github/workflows/deploy-supabase.yml`) runs `db push` + deploys both functions on merge to `main`, path-filtered to `supabase/**`, so UI-only PRs don't spend CI minutes.

## Architecture (the parts that span files)

- **Routing** ‚Äî `expo-router` v6, file-based under `src/app/`. `src/app/index.tsx` redirects on `authStore.session` to either `(onboarding)` or `(home)/(tabs)/team`. Two route groups: `(onboarding)` (signin/signup/connect-team/reset flows) and `(home)` (post-auth tabs: team / top-picks / transfer, plus player/profile/settings). `typedRoutes` and `reactCompiler` are both enabled experiments in `app.config.ts` ‚Äî **React Compiler is on, so don't hand-roll `useMemo`/`useCallback`/`React.memo` for memoization** unless profiling proves it's needed.

- **Auth gating** ‚Äî `src/lib/useProfileGate.ts` resolves a session into one of `loading | pending_deletion | missing | complete` (checks `profiles` + `account_deletions` tables; `pending_deletion` wins). Onboarding routes branch on this status. `src/app/_layout.tsx` holds the splash screen until fonts **and** auth/theme stores have hydrated, and wraps the tree in `QueryClientProvider`, `SafeAreaProvider`, `AuthErrorBoundary`, and `AuthCacheClear`.

- **Data layer** ‚Äî TanStack Query owns all server state. The two HTTP egress points are the only places network calls happen:
  - `src/api/fpl-client.ts` ‚Äî `fplGet()` wraps the public FPL API with timeout + retry (4xx never retried, 5xx backs off; throws `FplFetchError`). Swapping in a future `fpl-proxy` Edge Function only changes the base URL here.
  - `src/lib/supabase.ts` ‚Äî the `@supabase/supabase-js` singleton (AsyncStorage-backed session).
  - Everything in `src/api/*` is a hook or fetch fn; UI never calls Supabase/FPL directly. **Cache keys live only in `src/api/queryKeys.ts` ‚Äî never hand-roll a key array**, or invalidation breaks. FPL player ids are strings in our maps (`byId.get(String(pick.element))`); the "squad" is `/entry/{id}/event/{gw}/picks/` joined against `usePlayers()`.

- **Reads only today; FPL owns mutable team state** ‚Äî the app currently only *reads* FPL. Transfers, chip activation, captaincy, and lineup/bench order are **deferred to Phase 6**, gated on the user connecting their FPL account (FPL has no public write API or OAuth ‚Äî see issues #23‚Äì#27). When built, writes go through a **stateless `fpl-proxy` Edge Function** to FPL's authenticated endpoints (`my-team/{entry}`, `transfers`) with **FPL as the source of truth ‚Äî do not persist transfers/picks/chips in our Postgres** (the earlier `squads`/`transfers`/`chips_used` write design is abandoned). Our DB holds only auth/profile rows plus ingested reference data (clubs/players/fixtures).

- **Client state** ‚Äî Zustand stores in `src/store/` (`auth`, `biometric`, `team`, `theme`), persisted via AsyncStorage. Keep them narrow: anything server-derived belongs in React Query, not a store.

- **Theming** ‚Äî there is **no Tailwind/NativeWind** here (the lone `src/global.css` only declares web font variables). Styling runs off a custom token system: `src/constants/theme.ts` defines a `Theme` shape and three palettes (`classic | pitch | electric`), selected via `themeStore`. Companion tokens in `constants/apexTokens.ts`, `clubColors.ts`, `jerseys.ts`. Fonts are Archivo (display) + JetBrains Mono, loaded at the root layout.

- **Env vars** ‚Äî `EXPO_PUBLIC_SUPABASE_URL` / `EXPO_PUBLIC_SUPABASE_ANON_KEY` are read **only** in `app.config.ts`, which forwards them into Expo's `extra`. App code reads them via `Constants.expoConfig?.extra?.*` (see `src/lib/supabase.ts`). Reading `process.env.EXPO_PUBLIC_*` from app code yields `undefined` at runtime. Copy `.env.example` ‚Üí `.env` for local dev.

- **Backend** ‚Äî Supabase Postgres + RLS for authz, Deno Edge Functions for server logic, `pg_cron` for the FPL ingest. `supabase/migrations/` is the source of truth ‚Äî **never edit an applied migration; add a new timestamped one.** `supabase/functions/**` is a separate Deno toolchain: it's excluded from `tsconfig.json` and ignored by Jest ‚Äî don't run repo TS/lint/test tooling against it. Cron-driven ingest needs secrets seeded once per environment via `vault.create_secret` (the exact SQL is in `docs/architecture.md`).

- **Path aliases** ‚Äî `@/*` ‚Üí `src/*`, `@/assets/*` ‚Üí `assets/*` (in `tsconfig.json`, mirrored in the Jest `moduleNameMapper`).
